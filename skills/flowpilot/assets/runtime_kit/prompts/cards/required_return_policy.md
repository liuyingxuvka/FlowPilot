Formal role outputs must be written to run-scoped packet, result, report, decision, or blocker files and submitted with `flowpilot_runtime.py submit-output-to-router`; chat may contain controller-visible envelope metadata only. If an output contract has a fixed Router event, a local receipt or `submit-output` record is only local storage and must not be treated as wait completion until `submit-output-to-router` records the Router event.
