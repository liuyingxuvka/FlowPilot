## Context

The current formal runtime has a good mechanical path: `lease-agent`, `ack`, and `submit-result`. The missing piece is that Controller has no canonical, safe way to tell a newly opened role how to ACK, open its assigned packet, and submit without leaking sealed content. The role also has no obvious `flowpilot_new.py` command for opening the current packet body, even though a lower-level role-only opener exists.

## Goals / Non-Goals

**Goals:**

- Make packet handoff text generated by FlowPilot, not hand-written by Controller.
- Make the handoff generic for all runtime responsibilities.
- Add a formal `flowpilot_new.py open-packet` command that checks active lease, packet assignment, responsibility, ACK, body hash, and accepted-packet state before returning the sealed body to the role.
- Persist a sealed-body-open event and safe receipt metadata when the open succeeds.
- Add tests and FlowGuard evidence for the historical failure class: ACK succeeds, body is not in the ACK response, and the role must still have a formal way to open the packet.
- Keep Controller public status and handoff text body-free.

**Non-Goals:**

- Do not let Controller open sealed packet bodies.
- Do not give roles access to sibling packets, old-run packets, or other roles' results.
- Do not restore `controller_relay` as a required protocol.
- Do not migrate old run artifacts in place.

## Design

1. Add a small `flowpilot_core_runtime.role_handoff` helper.

   The helper renders one Controller-safe message from ledger facts: root path, script path, run id, packet id, lease id, responsibility, host kind, ACK command, open command, and submit command. It includes the old working authority sentence in generic form: the addressed role may open only the currently assigned packet through the runtime command and must not expose sealed content in chat.

2. Add `open-packet` to `flowpilot_new.py`.

   The command loads the current run ledger, checks the requested lease and packet, requires `ack_received`, calls the role-only packet opener, saves the ledger, and returns a role-facing JSON payload containing the sealed body plus safe packet metadata. The handoff tells roles to run this command themselves; Controller must not run it.

3. Strengthen role-only opener checks.

   The lower-level opener requires:

   - packet assigned to the lease;
   - lease active;
   - lease packet id matches;
   - lease responsibility matches packet envelope responsibility;
   - ACK already received;
   - packet not already accepted or quarantined;
   - current body hash matches the envelope.

4. Surface handoff from `lease-agent`.

   `lease-agent` returns a `role_handoff` object and `role_handoff_text` so Controller can paste a generated template without improvising. A separate `role-handoff` command can re-render the same safe text if needed.

5. Validate with model and tests.

   Add ordinary unit tests for all supported responsibilities and rejection cases. Add a focused FlowGuard check for the role packet access model: safe path reaches open, bad paths detect missing ACK, wrong lease, wrong role, stale/accepted packet, hash mismatch, body leak in handoff, and ad hoc narrow prompt.
