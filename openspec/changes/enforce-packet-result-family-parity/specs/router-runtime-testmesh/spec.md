## ADDED Requirements

### Requirement: Router runtime TestMesh exposes packet-result family evidence
The router runtime tier SHALL expose a focused packet-result family evidence
surface covering durable result reconciliation for material scan, research,
current-node, and PM role-work.

#### Scenario: Packet-result family child suite is selected
- **WHEN** the router runtime tier or focused packet validation is run
- **THEN** the packet-result family tests are discoverable as a distinct evidence surface
- **AND** the evidence surface identifies the covered packet-result families and mechanisms.

#### Scenario: Parent confidence consumes child family evidence
- **WHEN** a parent router runtime confidence claim includes packet-result reconciliation
- **THEN** the claim MUST consume current passing packet-result family evidence
- **AND** missing child evidence blocks or scopes the parent claim.
