# Architecture Reduction Decision Record

## Observable contract

FlowPilot has one high-standard, long-project route. Runtime owns mechanical
identity and current-contract validity; PM owns product and route decisions;
substantive roles complete bounded workstreams; independent FlowGuard and
Reviewer checks remain visible; final claims require current evidence.

## State and cleanup rule

No new durable state family is required. The discovery packet's local
inventory is packet-only input. The accepted discovery record stores only PM's
current candidate-skill selection and normal identity/timestamps. Workstream
plan rows live inside the role-authored semantic self-check/report body and
have no Runtime commit point. Removed material fields have terminal disposition
`forbidden_current_contract`; historical source labels may remain only in
explicit historical diagnostics or negative tests. The same terminal
disposition applies to `packet_reissue` and replacement-packet fields; they do
not retain a positive prompt, schema, model, or test route.

## Removed-name residue audit

The current runtime and schema occurrences of `material_sources`,
`material_sufficiency`, and `material_current` are rejection/forbidden-field
owners only. Retired card, packet, event, repair, and template names appear only
in negative tests, known-bad model scenarios, deleted lists, or explicit
historical labels. Generic source-evidence fields such as a Reviewer's list of
checked source material remain current because they do not create a dedicated
material phase or result authority. Stale positive documentation in
`docs/parallel_packet_batch_plan.md` and
`docs/flowpilot_process_contract_binding_plan.md` was replaced with the
existing research/current-node/PM-role-work ownership contract.
