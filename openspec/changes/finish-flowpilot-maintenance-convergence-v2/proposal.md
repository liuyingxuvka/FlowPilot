## Why

FlowPilot now has a FlowGuard-first work-order kernel, many completed local
maintenance changes, and a non-green coverage inventory that separates stale
validation boundaries from remaining runtime, protocol, terminal, structure,
model, and test evidence debt. The next pass must converge those residuals
without rewriting the system, hiding failed evidence, breaking compatibility
facades, or committing unverified peer-agent work.

## What Changes

- Add a final maintenance-convergence gate that drives OpenSpec scope,
  FlowGuard model ownership, implementation slices, background regressions,
  local install sync, and local git finalization as one evidence-bound process.
- Repair remaining FlowGuard coverage gaps by class first: protocol/source
  conformance, terminal monotonicity, live runtime state, final confidence,
  model/test tier classification, and behavior-preserving structure hotspots.
- Preserve public imports, CLI behavior, event names, persisted JSON shapes,
  prompt/control semantics, and current runtime evidence unless a scoped bug
  fix explicitly requires a behavior repair.
- Run heavyweight model regressions through the repository background log
  contract and inspect final stdout, stderr, combined, exit, and meta artifacts
  before claiming pass.
- Sync the repository-owned installed FlowPilot skill, audit install freshness,
  validate OpenSpec and FlowGuard evidence, and create a local git commit
  without push, tag, deploy, release, or remote mutation.

## Capabilities

### New Capabilities

- `flowpilot-maintenance-convergence-v2`: Final end-to-end maintenance
  convergence for remaining FlowGuard coverage gaps, structure/test/model
  compression, install sync, and local git finalization.

### Modified Capabilities

- `behavior-preserving-structural-maintenance`: Consume the v2 convergence gate
  before broad module contraction can be considered complete.
- `flowguard-full-model-coverage-inventory`: Treat residual v2 runner groups as
  explicit convergence blockers until each has direct current evidence.
- `final-structure-convergence`: Require the v2 finalization sequence before
  local structure convergence can claim install and git synchronization.

## Impact

- Affected code: targeted `skills/flowpilot/assets/`, `simulations/`, `scripts/`,
  and `tests/` surfaces that own the residual FlowGuard gaps.
- Affected OpenSpec files: this new change plus spec deltas for the v2
  convergence gate and existing maintenance/finalization requirements.
- Affected evidence: maintenance map, coverage sweep, full coverage inventory,
  protocol conformance, terminal monotonicity, control-plane friction, final
  confidence, model/test alignment, TestMesh/StructureMesh evidence, background
  Meta/Capability regressions, install sync/audit/check, and local git scope.
- Out of scope: remote push, release publication, destructive runtime cleanup,
  broad rewrite, dependency stack changes, or silent acceptance-contract changes.
