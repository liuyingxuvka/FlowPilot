## ADDED Requirements

### Requirement: Cartesian obligations bind model, owner code, and external execution
Every required risk shard SHALL bind a FlowGuard model obligation to its owner
`CodeContract`, source audit, public-runtime or boundary observation, and
current external-contract test evidence.

#### Scenario: Expected metadata is copied as observed output
- **WHEN** an alignment row derives observed owner, reaction, status, or side
  effects directly from expected model data
- **THEN** the row MUST fail source/runtime evidence review
- **AND** it MUST not support a binding-row closure claim

#### Scenario: A broad obligation hides leaf risk shards
- **WHEN** one coarse obligation represents multiple identity, branch, timing,
  review, repair, closure, or side-effect shards
- **THEN** MTA MUST split or project stable shard obligations
- **AND** every required shard MUST cite its own current evidence target

#### Scenario: Resume evidence binds only the retained diagnostic router
- **WHEN** a resume obligation cites internal router tests but does not bind the
  public `flowpilot_new.py resume` and `dispatch-current-role` behavior
- **THEN** MTA MUST keep public requested-role resume unclosed
- **AND** diagnostic resume evidence MUST NOT substitute for the public
  CodeContract or external-contract test

#### Scenario: Resume target-set obligation is modeled
- **WHEN** MTA registers current-run resume coverage
- **THEN** it MUST bind inclusion of every current obligation owner, exclusion
  of idle/history/fixed-role slots, immediate foreground-recipient handling,
  target deduplication, current binding identity, and exact recovery behavior
- **AND** positive and known-bad cases MUST use current public and diagnostic
  observations without treating either as a second authority

#### Scenario: Reviewer model requires a deleted output field
- **WHEN** a Reviewer model, fake-AI fixture, CodeContract, or positive test
  treats `independent_challenge` as required successful payload
- **THEN** MTA MUST report a field-lifecycle and model-contract mismatch
- **AND** closure MUST require compact result shape plus semantic challenge
  evidence through existing fields

#### Scenario: Workstream rows are treated as semantic proof
- **WHEN** an alignment row infers substantive completion only from the
  presence of `Workstream Plan and Completion`
- **THEN** MTA MUST reject that observation as insufficient
- **AND** the Reviewer evidence target MUST bind real artifact, delegation,
  verification, unresolved-item, and claim-consistency review

### Requirement: Real miss backfeed updates the whole evidence chain
Every failure found by public execution SHALL map to its existing behavior
commitment/owner, contract mutation case, same-class cases, runtime regression,
MTA row, TestMesh shard, and parent receipt before the miss is closed.

#### Scenario: Runtime fails after model coverage passed
- **WHEN** a selected public-pipeline case contradicts the model or oracle
- **THEN** the finding MUST enter Model-Miss Review and ObservedProblemBackfeed
- **AND** a later local green test alone MUST NOT close release confidence
