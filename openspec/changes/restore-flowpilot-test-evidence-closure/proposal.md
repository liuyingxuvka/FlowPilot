## Why

FlowPilot already generates very large contract-exhaustion, fake-AI, and
Cartesian matrices, but current evidence can still report those generated
cells as passed without proving that the real `ACK -> open-packet ->
submission_checklist -> submit-result` path executed.  The audit also found a
vacuous forbidden-field lane: thirteen base result families register 122
forbidden fields while the fake responder currently projects zero of them.

This change turns the existing broad coverage plan into execution-backed,
time-bounded evidence without pretending that every natural-language AI output
or every unconstrained multi-axis product can be run end to end.

The follow-up authority audit also found two current-contract convergence
gaps.  Public manual resume is requested-role driven while the retained
diagnostic router can still infer a wider same-run role set, and one handoff
description/model still treats the deleted `independent_challenge` object as a
positive Reviewer field.  These are repaired through the existing
foreground-duty, node-context, workstream, compact review, and evidence-mesh
owners rather than a new role system or result protocol.

## What Changes

- **BREAKING** Make the envelope `current_handoff_contract` the only
  mechanical result-contract authority.  Packet bodies may provide semantic
  task material but may no longer override or repair the submission checklist.
- **BREAKING** Make fake-AI rehearsal consume the real current
  `open-packet` result and submit through the public `submit-result` surface;
  remove body/private-helper/generic-family success paths.
- Project forbidden fields, forbidden aliases, branch shapes, option/type
  rules, current identities, review policy, and a deterministic contract
  fingerprint into the role-visible checklist.
- Declare a finite FlowGuard `ContractCoverageUniverse`, explicit exclusions,
  executable oracles, unique case ids, and equivalence rules for any many-to-one
  canonicalization.
- Separate full structural enumeration from execution coverage: all single
  mechanical faults execute, a constraint-aware pairwise set traverses the
  public pipeline, high-risk interactions receive three-wise coverage, and all
  historical misses plus deterministic fuzz cases remain mandatory.
- Replace hard-coded `passed/current` TestMesh rows and test-name reuse with
  proof-backed shard results, current fingerprints, MTA owner bindings, and
  parent receipt consumption.
- Register explicit PR, nightly, release, and final-confidence tiers.  Long
  regressions run in bounded background supervisors, where progress is
  liveness only and final exit/result artifacts are required for pass.
- Treat `daemon_replay`, legacy responsibility aliases, body-only contracts,
  and private-helper replay as historical/negative surfaces only.
- Require one structured current-authority reference shape inside existing
  handoff/context reference surfaces so original user intent, accepted
  requirements, active route/node/packet identity, and current evidence remain
  fingerprinted and machine-auditable instead of being recovered from prose or
  chat history.
- Make manual resume restore or replace exactly the roles demanded by current
  unresolved packet obligations plus the immediate foreground duty.  Idle
  same-run slots, fixed role sets, prior-run bindings, and historical memory
  are not positive restoration authority.
- Preserve the existing `Workstream Plan and Completion` structure for every
  substantive role while keeping semantic completeness, evidence quality, and
  claim consistency with Reviewer.  Reviewer results remain on the compact
  current contract; `independent_challenge` stays deleted and forbidden as a
  result field.
- Require timeout, cancellation, or interruption cleanup to prove zero live
  descendants before any evidence is accepted or another execution owner
  starts.
- Canonicalize controlled text inputs to LF and retain one frozen snapshot
  fingerprint as provenance and mixed-snapshot protection.  The snapshot
  fingerprint is not evidence-applicability authority and does not authorize a
  blanket rerun.
- Derive each execution owner's covered-input identity from the existing
  MTA/TestMesh ownership map.  Reuse unaffected terminal proof through a
  current `TestResultReuseTicket`, execute only mapped affected owners and
  their declared dependents, execute one declared full owner only for an
  explicitly mapped shared-global input, and block `impact_mapping_missing`
  rather than falling back to `run all`.
- Enforce direct current replacement with zero compatibility, translation,
  legacy alias, fallback authority, or second success path.
- Supersede the closure claims of earlier incomplete Cartesian/fake-AI changes
  while retaining their requirements, historical fixtures, and already
  verified control-plane ledger repairs through an explicit predecessor
  disposition table.

## Capabilities

### New Capabilities

- `flowpilot-ai-response-cartesian-execution-closure`: Defines the finite
  AI-response coverage universe, tiered execution lanes, executable oracles,
  proof receipts, and qualified `full within declared universe` claim.

### Modified Capabilities

- `flowpilot-control-plane-contract-kernel`: Makes the envelope handoff the
  single mechanical authority, requires structured current-authority
  references and exact requested-role resume targets, and forbids
  body/private/daemon/fixed-role alternate success paths.
- `flowpilot-packet-review-flow`: Requires Reviewer rehearsal to consume the
  delivered review window and review-depth policy, keeps the existing
  workstream structure semantically Reviewer-owned, and preserves the compact
  result contract with the deleted challenge object forbidden.
- `synthetic-agent-coverage-matrix`: Separates generated, selected, executed,
  passed, excluded, stale, and proof-backed counts and closes forbidden-field
  cardinality.
- `end-to-end-synthetic-agent-chaos-replay`: Requires public current-runtime
  packet execution rather than body-derived or private-helper replay.
- `flowguard-test-obligation-ownership`: Requires real child-suite evidence,
  shard receipts, same-fingerprint freshness, descendant-zero cleanup, and
  parent consumption.
- `flowguard-boundary-test-alignment`: Binds generated obligations to owner
  code contracts, public-runtime observations, source audit, external tests,
  exact resume-target obligations, and compact Reviewer semantics.
- `tiered-flowpilot-test-validation`: Adds explicit formal-submit fast,
  adversarial/nightly, release, and final-confidence ownership, budgets,
  descendant cleanup, and one-fingerprint sequencing.
- `flowpilot-maintenance-ideal-state`: Makes installation sync, versioning,
  local candidate handoff, predecessor disposition, evidence freshness, and
  the now-authorized Git/GitHub publication part of one closure.

## Impact

- Runtime and role surfaces:
  `flowpilot_core_runtime/runtime.py`, `packet_result_contracts.py`,
  `role_handoff.py`, `host.py`, `flowpilot_new_role_commands.py`, public resume
  and dispatch, and the retained diagnostic resume-role projection.
- Fake-AI and replay surfaces: contract-driven responder, fake project CLI,
  current core rehearsal, Cartesian/contract-exhaustion models, and public
  submit harnesses.
- FlowGuard evidence: ContractExhaustionMesh, FieldLifecycleMesh,
  Model-Test Alignment, TestMesh, ModelMesh receipts, topology, Meta, and
  Capability parents.
- Test orchestration: focused PR tests, deterministic pairwise/high-risk
  shards, exact requested-role and compact-review known-bad cases, background
  nightly/release tiers, descendant-zero cleanup evidence, and proof artifacts
  bound to one covered-source fingerprint.
- Local candidate surfaces: installed Codex skill, VERSION, changelog, README
  release statements, final local verification, one exact task-owned local Git
  commit, fast-forward default-branch publication, one annotated `v0.12.1`
  tag, and one source-only GitHub Release, all bound to the same commit.

The change retains the verified behavior fixes and historical fixtures from
`harden-flowpilot-control-plane-ledger-hygiene`.  It supersedes only broad
execution-coverage and release-closure claims that were previously supported
by generated cells, abstract-green rows, or incomplete OpenSpec changes.
