## ADDED Requirements

### Requirement: Real Router rehearsal consumes executable bridge rows

Real Router dry-run rehearsal SHALL cite executable bridge rows when claiming
prepared fake AI packages exercised public runtime boundaries.

#### Scenario: Rehearsal row binds CLI evidence

- **WHEN** a real Router rehearsal row claims public Runtime/CLI coverage
- **THEN** it MUST cite the bridge row, public CLI commands, event-log evidence,
  result path, and freshness receipt for the same expected outcome

#### Scenario: CLI worker lifecycle is covered

- **WHEN** a rehearsal uses persistent public CLI workers or one-shot commands
- **THEN** it MUST record worker lifecycle expectations, including one-shot
  command classes, command budget, restart behavior, timeout handling, and
  cleanup evidence

### Requirement: Open-packet uses sealed packet authority

Real Router rehearsal SHALL prove `open-packet` obtains authorized sealed packet
inputs rather than public result projections or stale result paths.

#### Scenario: Open-packet reads authorized sealed body

- **WHEN** a rehearsal opens the current packet for a role lease
- **THEN** the public CLI evidence MUST show the packet id and lease id matched
  the current authority and the returned body came from the sealed packet body
