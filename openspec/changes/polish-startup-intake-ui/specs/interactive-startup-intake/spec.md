## ADDED Requirements

### Requirement: Startup intake dialog preserves workflow while presenting polished visual proportions
The native startup intake dialog SHALL preserve the existing single-screen startup workflow while using restrained visual proportions, neutral ordinary chrome, and clear primary-action copy.

#### Scenario: Polished dialog keeps the same intake behavior
- **WHEN** the native startup intake dialog is opened for formal startup
- **THEN** it still presents the work-request field, background-collaboration control, settings entry, and confirmation action without adding a new workflow step or changing result artifact schemas

#### Scenario: Visual polish remains scoped
- **WHEN** the startup intake dialog is visually refreshed
- **THEN** language and developer-support controls remain in the settings popup, ordinary borders use neutral styling, and the brand accent is reserved for primary, enabled, hover, or focus states
